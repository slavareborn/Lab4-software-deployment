# Автоматизоване розгортання веб-інфраструктури (IaC)

Цей проєкт реалізує автоматизоване розгортання та налаштування розподіленої інфраструктури за допомогою **Terraform** та **Ansible**. Система складається з двох віртуальних машин, що працюють у зв'язці для забезпечення роботи веб-застосунку.

---

## 🏗 Архітектура системи

Проєкт базується на моделі **"Web-Server + Database Server"**:

* **Worker VM (Web Server)** — хостить Node.js застосунок та Nginx, який працює як реверс-проксі
* **Database VM** — хостить PostgreSQL, ізольовану від зовнішнього доступу

---

## 🛠 Технологічний стек

* **Інфраструктура:** Terraform, VirtualBox
* **Конфігурація:** Ansible
* **Застосунок:** Node.js, PostgreSQL, Nginx

---

## 🚀 Інструкція з розгортання

### 1. Підготовка інфраструктури (Terraform)

Перейдіть до директорії з Terraform-файлами:

```bash
cd terraform
```

Ініціалізуйте та застосуйте конфігурацію:

```bash
terraform init
terraform apply
```

Після успішного розгортання отримайте IP-адреси створених віртуальних машин та внесіть їх у файл `ansible/inventory.ini`.

---

### 2. Налаштування серверів (Ansible)

Перейдіть до директорії з Ansible-проєктом:

```bash
cd ../ansible
```

Запустіть основний плейбук:

```bash
ansible-playbook -i inventory.ini site.yml
```

---

## ⚙️ Особливості налаштування

* **Ідемпотентність:**
  Плейбуки Ansible можна запускати багаторазово без ризику пошкодження конфігурації

* **Безпека:**

  * Налаштовано права доступу користувачів `app` та `operator`
  * PostgreSQL приймає з'єднання лише з довіреного IP worker-вузла

* **Reverse Proxy:**
  Nginx проксіює запити до Node.js, забезпечуючи захист та кешування

---

## ✅ Перевірка працездатності

Після розгортання перевірте доступність застосунку за IP вашої **worker-vm**:

| Ендпоінт        | Опис                   | Очікуваний результат |
| --------------- | ---------------------- | -------------------- |
| `/health/alive` | Перевірка процесу      | `alive`              |
| `/health/ready` | Перевірка зв'язку з БД | `ready`              |
| `/`             | Головна сторінка       | Вітання з варіантом  |

## 📝 Troubleshooting (Вирішення проблем)

### ❌ Помилка 502 Bad Gateway

* Переконайтеся, що Node.js запущений:

```bash
systemctl status webapp
```

* Перевірте відповідність порту в `server.js` та конфігурації Nginx

---

### ❌ Помилка підключення до БД

* Перевірте правильність IP-адреси бази даних у файлі:

```
/opt/webapp/server.js
```
