# Лабораторна робота №1: Розгортання Web-сервісу з автоматизацією

## Варіант
- **Тема застосунку:** Simple Inventory (сервіс обліку обладнання)
- **Порт застосунку:** 8080
- **База даних:** MariaDB
- **Номер варіанту:** 20

## Архітектура системи
Система розгортається на одній віртуальній машині Linux та складається з наступних компонентів:
`client → nginx (reverse proxy) → Node.js web application → MariaDB`

Мережеві обмеження:
- Nginx слухає на порту 80 (доступний ззовні).
- Web app слухає на `127.0.0.1:8080` (доступний лише локально).
- MariaDB слухає на `127.0.0.1:3306` (доступна лише локально).

## Документація по API
Сервіс підтримує Content Negotiation (заголовки `Accept: text/html` або `Accept: application/json`).

| Метод | Ендпоінт | Опис |
| :--- | :--- | :--- |
| GET | `/` | Повертає HTML сторінку зі списком доступних шляхів (тільки HTML) |
| GET | `/items` | Отримати список усіх предметів (id, name) |
| POST | `/items` | Створити новий запис (тіло запиту JSON: `{"name": "Item", "quantity": 10}`) |
| GET | `/items/<id>`| Вивести детальну інформацію про запис |
| GET | `/health/alive`| Перевірка працездатності сервісу (завжди повертає 200 OK) |
| GET | `/health/ready`| Перевірка підключення до БД (повертає 200 OK або 500) |

*Примітка: Ендпоінти `/health/*` недоступні ззовні через Nginx.*

## Інструкція з розгортання

### 1. Вимоги до середовища
- **ОС:** Ubuntu Server 22.04 LTS або 24.04 LTS (офіційний образ).
- **Ресурси:** Мінімум 1 vCPU, 1 GB RAM, 10 GB Disk.
- **Доступ:** SSH доступ до віртуальної машини з правами `sudo`.

### 2. Запуск автоматизації
1. Склонуйте цей репозиторій на віртуальну машину.
2. Перейдіть до папки проєкту: `cd Lab1-software-deployment`
3. Зробіть скрипт виконуваним: `chmod +x deploy/install.sh`
4. Запустіть скрипт від імені суперкористувача: `sudo ./deploy/install.sh`

### 3. Доступ до системи після розгортання
Скрипт автоматично блокує дефолтного користувача (напр. ubuntu) та створює нових:
- **student** — з повними правами sudo (вхід по ключу або без пароля, якщо налаштовано).
- **teacher** — пароль за замовчуванням `12345678` (вимагає зміни при першому вході), має права sudo.
- **operator** — пароль `12345678` (вимагає зміни), має право виконувати лише команди керування сервісом `mywebapp` та `nginx` через sudo.

### 4. Тестування
Щоб перевірити коректність роботи:
1. Відкрийте браузер або використайте `curl`: `curl http://<IP_ВІРТУАЛЬНОЇ_МАШИНИ>/items`
2. Щоб додати елемент: `curl -X POST -H "Content-Type: application/json" -d '{"name":"Laptop", "quantity":5}' http://<IP_ВІРТУАЛЬНОЇ_МАШИНИ>/items`
